# 🛡️ SOC Technical Investigation: RDP Bruteforce
**Lead Investigator:** Harvey R  
**Academy:** LearnCyber Academy - Intermediate Capstone

---

## 1. Executive Attack Summary
The organization's Windows environment was targeted by a high-intensity, automated **Remote Desktop Protocol (RDP)** bruteforce attack. Forensic analysis identified a single source IP address, **113.161.192.227**, originating from **Vietnam**, which attempted **3,103** unauthorized logins against the local 'administrator' account. The attack utilized **Network Logon (Type 3)** via the NTLM protocol, suggesting the use of automated password-spraying tools. While thousands of failures were recorded, no successful compromise (Event ID 4624) was observed from the attacker's IP.

---

## 2. Detection Source
The suspicious activity was detected through a manual forensic review of the **Windows Security Event Logs** generated by the `Microsoft-Windows-Security-Auditing` service.

## 4. The Attack Chain Breakdown
The attacker followed a standard automated reconnaissance and access pattern.

| Stage | Observation | Technical Detail |
| :--- | :--- | :--- |
| **Entry Point** | **External RDP Access** | Targeted the default 'administrator' account on Port 3389. |
| **Execution** | **Credential Spraying** | 3,103 attempts using ephemeral source ports (49162–65534). |
| **Persistence** | **Failure to Access** | No successful logon (4624) achieved; access was blocked by password requirements. |
| **Lateral Movement**| **None Detected** | Activity was confined to a single host; no evidence of movement to other systems. |
| **Exfiltration** | **No Data Loss** | Credential Manager access (5379) was performed by SYSTEM, not the attacker. |

## 7. Tools Utilized
The forensic analysis was completed using the following toolset:
*   **Log Analysis:** **Microsoft Excel** and **Sublime Text** for triaging CSV-exported logs.
*   **Endpoint Forensics:** **Windows Event Viewer** for EVTX log analysis.
*   **Reconnaissance:** **IP Geolocation** to attribute the source IP to Vietnam.

---

## 8. Key Findings & Weakness Summary
*   **Findings:** While the attack failed to gain entry, the volume of attempts proves that the organization's perimeter is being actively scanned by automated botnets.
*   **Key Weakness:** The critical lack of an **Account Lockout Policy** allowed the attacker to make over 3,000 attempts without being blocked. Additionally, the exposure of **Port 3389 (RDP)** directly to the internet without a VPN is a major policy failure.
